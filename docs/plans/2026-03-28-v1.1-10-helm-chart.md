# v1.1 Helm Chart — Technická analýza

## Popis

Kubernetes Helm chart pro nasazení Shieldoo Gate. Umožní one-command deployment na Kubernetes s konfigurovatelným storage, databází, ingress a scaling.

### Proč

- Kubernetes je standard pro produkční deployment
- Docker Compose je vhodný pro dev/test, ne pro produkci
- `initial-analyse.md:920-929` definuje Helm deployment
- Enterprise zákazníci očekávají Helm chart

### Out of Scope

- Helm chart registry (publikace na charts.shieldoo.io) — budoucí
- Operator pattern — pouze Helm chart
- Auto-scaling (HPA) — manuální scaling v v1.1

## Aktuální stav

### Docker setup

`docker/Dockerfile`:
- Multi-stage build: Node (UI) → Go (binary) → Trivy → Alpine runtime
- Non-root user `sgw` (řádky 38-40)
- Porty: 5000 (PyPI), 4873 (npm), 5001 (NuGet), 5002 (Docker), 8080 (Admin)

`docker/docker-compose.yml`:
- Volumes: bridge-socket, gate-data, gate-cache, trivy-cache, config
- Scanner bridge sidecar
- Push registry sidecar (optional)

### Potřebné Kubernetes resources

| Resource | Účel |
|----------|------|
| Deployment | shieldoo-gate main container + scanner-bridge sidecar |
| Service | ClusterIP pro interní přístup |
| Ingress | External přístup (optional) |
| ConfigMap | config.yaml |
| Secret | OIDC secrets, DB credentials, webhook secrets |
| PVC | gate-data (SQLite), gate-cache (local cache) |
| ServiceAccount | pro cloud storage IAM (optional) |

### values.yaml (klíčové sekce)

```yaml
replicaCount: 1  # >1 vyžaduje PostgreSQL + cloud storage

image:
  repository: ghcr.io/cloudfieldcz/shieldoo-gate
  tag: ""  # defaults to chart appVersion
  pullPolicy: IfNotPresent

ports:
  pypi: 5000
  npm: 4873
  nuget: 5001
  docker: 5002
  admin: 8080
  # maven: 5003  (pokud v1.1 adaptéry enabled)

service:
  type: ClusterIP
  metricsPort: 9090  # separátní port pro /metrics (viz v1.1-05)

ingress:
  enabled: false
  className: ""
  annotations: {}
  hosts:
    - host: shieldoo-gate.example.com
      paths:
        - path: /
          pathType: Prefix
          port: 8080  # admin

persistence:
  data:
    enabled: true
    size: 1Gi
    storageClass: ""
  cache:
    enabled: true
    size: 50Gi
    storageClass: ""

scannerBridge:
  enabled: true
  image:
    repository: ghcr.io/cloudfieldcz/shieldoo-scanner-bridge
    tag: ""

config: {}  # inline config.yaml overrides

existingSecret: ""  # name of pre-existing Secret

resources:
  limits:
    memory: 2Gi
  requests:
    cpu: 500m
    memory: 512Mi
```

### Sidecar pattern

Scanner bridge jako sidecar kontejner v Deployment (shared Unix socket via emptyDir volume):

```yaml
containers:
  - name: shieldoo-gate
    ...
    volumeMounts:
      - name: bridge-socket
        mountPath: /var/run/shieldoo/
  - name: scanner-bridge
    image: "{{ .Values.scannerBridge.image.repository }}"
    volumeMounts:
      - name: bridge-socket
        mountPath: /var/run/shieldoo/
volumes:
  - name: bridge-socket
    emptyDir: {}
```

### NetworkPolicy (optional)

```yaml

apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: {{ include "shieldoo-gate.fullname" . }}
spec:
  podSelector:
    matchLabels: {{ include "shieldoo-gate.selectorLabels" . | nindent 6 }}
  ingress:
    - ports:
        - port: {{ .Values.ports.admin }}
      from:
        - namespaceSelector:
            matchLabels:
              name: {{ .Values.networkPolicy.adminNamespace }}
```

### PodDisruptionBudget (pro HA)

```yaml
# templates/pdb.yaml — podmíněně pokud replicaCount > 1
{{- if gt (int .Values.replicaCount) 1 }}
apiVersion: policy/v1
kind: PodDisruptionBudget
metadata:
  name: {{ include "shieldoo-gate.fullname" . }}
spec:
  minAvailable: 1
  selector:
    matchLabels: {{ include "shieldoo-gate.selectorLabels" . | nindent 6 }}
{{- end }}
```

### Secrets vs ConfigMap

**DŮLEŽITÉ:** Sensitive hodnoty (OIDC client secret, DB DSN, webhook secrets) MUSÍ být v Kubernetes `Secret`, NIKDY v `ConfigMap`. ConfigMap je uložen v cleartext v etcd.

```yaml
# templates/secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: {{ include "shieldoo-gate.fullname" . }}-secrets
type: Opaque
data:
  # Hodnoty z values.yaml nebo external secret management
```

Config template reference secret přes env vars, ne přes inline config:
```yaml
env:
  - name: SGW_DATABASE_POSTGRES_DSN
    valueFrom:
      secretKeyRef:
        name: {{ .Values.existingSecret | default (printf "%s-secrets" (include "shieldoo-gate.fullname" .)) }}
        key: database-dsn
```

### Image security

Podpora image pull by **digest** (ne jen tag):
```yaml
image:
  repository: ghcr.io/cloudfieldcz/shieldoo-gate
  tag: ""           # mutable — použít pro dev
  digest: ""        # immutable — preferovat pro produkci
  pullPolicy: IfNotPresent
```

### Chart-level HA validace

```yaml
# templates/_helpers.tpl — assertion
{{- if and (gt (int .Values.replicaCount) 1) (eq .Values.database.backend "sqlite") }}
  {{- fail "replicaCount > 1 requires database.backend=postgres (SQLite is single-writer)" }}
{{- end }}
{{- if and (gt (int .Values.replicaCount) 1) (eq .Values.cache.backend "local") }}
  {{- fail "replicaCount > 1 requires cache.backend=s3|azureblob|gcs (local cache is not shared)" }}
{{- end }}
```

## Akceptační kritéria

1. `helm install` s default values vytvoří fungující single-replica deployment.
2. `helm install` s PostgreSQL + S3 values vytvoří stateless multi-replica deployment.
3. `helm lint` projde bez warningů.
4. `replicaCount > 1` s SQLite/local cache → helm error (chart-level validace).
5. Všechny porty jsou exponovány přes Service.
6. Secrets jsou v Kubernetes Secret, nikdy v ConfigMap.
7. SecurityContext nastavuje runAsNonRoot, readOnlyRootFilesystem, no privilege escalation.
8. Liveness/readiness probes fungují proti /api/v1/health.
9. Instalace: `helm install shieldoo-gate ./helm/shieldoo-gate/`

## Poznámky

- **Instalace:** `helm install shieldoo-gate ./helm/shieldoo-gate/` (lokální path). Chart registry publishing (charts.shieldoo.io) je budoucí.
- **Mutable image tags:** Default `tag: ""` (appVersion) je mutable — stejný princip který tag mutability detection (v1.1-03) chrání. Pro produkci doporučit `digest:` field.
- **Dokumentační deliverables:** `docs/index.md`, `docs/deployment.md` (Helm section).
