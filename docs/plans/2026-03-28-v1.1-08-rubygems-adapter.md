# v1.1 RubyGems Adapter — Technická analýza

## Popis

Nový protokolový adaptér pro RubyGems (Ruby package manager). Implementuje RubyGems API jako transparent proxy s artifact scanning.

### Proč

- Ruby ekosystém — Rails je stále masivně používaný framework
- Supply chain útoky na RubyGems: typosquatting, backdoored gems
- `initial-analyse.md:596-599` definuje RubyGems adapter jako v1.1

### Out of Scope

- Gem push (upload) — pouze proxy/read
- Bundler API v2 (compact index) — **VAROVÁNÍ:** Moderní Bundler primárně používá compact index API (`/api/v1/dependencies`, `/info/{name}`), ne legacy specs. Bez compact index může `bundle install` být pomalý (fallback na full index). **Ověřit:** zda Bundler gracefully fallbackuje na legacy API. Pokud ne, přesunout compact index do Fáze 1.

## Návrh řešení

### RubyGems API Layout

```
GET /api/v1/gems/{name}.json                    ← gem metadata (JSON)
GET /api/v1/versions/{name}.json                ← version listing
GET /gems/{name}-{version}.gem                  ← download gem (trigger scan)
GET /quick/Marshal.4.8/{name}-{version}.gemspec.rz  ← compressed gemspec
GET /specs.4.8.gz                               ← full index (proxy pass-through)
GET /latest_specs.4.8.gz                        ← latest index
GET /prerelease_specs.4.8.gz                    ← prerelease index
```

### Routing

```go
r.Get("/api/v1/gems/{name}.json", a.handleGemInfo)
r.Get("/api/v1/versions/{name}.json", a.handleVersions)
r.Get("/gems/{name}-{version}.gem", a.handleGemDownload) // trigger scan
r.Get("/quick/Marshal.4.8/*", a.handleQuickSpec)          // pass-through
r.Get("/specs.4.8.gz", a.handleIndex)                     // pass-through
r.Get("/latest_specs.4.8.gz", a.handleIndex)              // pass-through
r.Get("/prerelease_specs.4.8.gz", a.handleIndex)          // pass-through
```

**Gem filename parsing — pozor na ambiguitu:**
Formát `{name}-{version}.gem` je nejednoznačný pro gemy s pomlčkami v názvu (např. `aws-sdk-core-3.0.0.gem`). Parsovat zprava: najít poslední pomlčku následovanou číslem pro oddělení name a version. Platform-specific gemy (e.g., `nokogiri-1.16.0-x86_64-linux.gem`) obsahují platformu za verzí.

## Akceptační kritéria

1. `gem install <package> --source http://shieldoo-gate:5004/` uspěje pro čistý gem.
2. Malicious gem je blokován s HTTP 403.
3. Metadata/index requesty jsou proxovány bez scanningu.
4. Gem s pomlčkami v názvu (e.g., `aws-sdk-core`) je správně parsován.
5. Platform-specific gem (e.g., `nokogiri-1.16.0-x86_64-linux.gem`) je správně parsován.
6. `EcosystemRubyGems` konstanta je unikátní (test pro kolizi s ostatními ekosystémy).

## Rizika a mitigace

| Riziko | Dopad | Pravděpodobnost | Mitigace |
|--------|-------|-----------------|----------|
| Bundler nepoužitelný bez compact index | Vysoký — hlavní Ruby dependency manager | Střední | Ověřit fallback; přidat compact index pokud nutné |
| Gem name parsing ambiguita | Střední — špatný artifact ID | Střední | Parsovat zprava; extensive unit testy |
| Platform-specific gem parsing | Nízký — nesprávný cache key | Nízká | Zahrnout platformu v artifact ID |

## Poznámky

- **Bundler kompatibilita** je klíčový business requirement. Před implementací ověřit: `bundle install --no-color 2>&1` s proxy nastaveným na Shieldoo Gate (legacy API only). Pokud Bundler selže, compact index je Fáze 1, ne out-of-scope.
- **Dokumentační deliverables:** `docs/index.md`, `docs/adapters.md`, `docs/configuration.md`.
