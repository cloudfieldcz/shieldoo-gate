# v1.1 Go Modules Adapter — Technická analýza

## Popis

Nový protokolový adaptér pro Go module proxy (GOPROXY protocol). Implementuje Go module proxy API jako transparent proxy s artifact scanning.

### Proč

- Go ekosystém roste — Go modules je standard od Go 1.13
- Supply chain rizika: typosquatting, dependency confusion na pkg.go.dev
- Shieldoo Gate je sám napsaný v Go — dogfooding use case
- `initial-analyse.md` implicitly lists Go modules among v1.1 adapters

### Out of Scope

- Module publishing — pouze proxy/read
- Checksum database validation (sum.golang.org) — budoucí. **Bezpečnostní omezení:** Go security model spoléhá na checksum DB. Bez ní Shieldoo Gate nemůže detekovat tampered moduly s validními checksums v proxy. Dokumentovat jako known security limitation. Doporučit klientům ponechat `GONOSUMDB` nenastavené.
- Private module authentication (GOPRIVATE, Git credentials) — out of scope pro v1.1.

## Návrh řešení

### Go Module Proxy Protocol (GOPROXY)

```
GET /{module}/@v/list           ← version listing (text, one per line)
GET /{module}/@v/{version}.info ← version metadata (JSON: {"Version": "v1.0.0", "Time": "..."})
GET /{module}/@v/{version}.mod  ← go.mod file (text)
GET /{module}/@v/{version}.zip  ← source zip (trigger scan)
GET /{module}/@latest           ← latest version info
```

Module path encoding: `github.com/Foo/Bar` → `github.com/!foo/!bar` (uppercase → `!` + lowercase)

**Implementace encoding/decoding:** Použít standardní knihovnu `golang.org/x/mod/module`:
- `module.EscapePath()` pro encoding
- `module.UnescapePath()` pro decoding
NIKDY implementovat custom encoding — edge cases jsou netriviální.

**Major version suffix:** Go module paths mohou mít major version suffix (e.g., `github.com/user/repo/v2`). Path parser musí správně zacházet s `/v2`, `/v3` etc. jako součástí module path, ne jako oddělený segment.

### Routing

```go
// POZOR: Chi {param} zachytí pouze jeden path segment.
// Go module paths obsahují lomítka (github.com/user/repo).
// Nutné použít catch-all route s manuálním parsováním.
r.Get("/*", a.handleRequest)  // catch-all, manuální path parsing

// handleRequest parsuje cestu a routuje:
// - /@v/list     → handleVersionList
// - /@v/*.info   → handleVersionInfo
// - /@v/*.mod    → handleGoMod (pass-through)
// - /@v/*.zip    → handleZipDownload (trigger scan)
// - /@latest     → handleLatest
```

**Chi `{module}` parametr NELZE použít** — zachycuje pouze jeden segment cesty (do prvního `/`). Všechny reálné module paths obsahují `/` (github.com/user/repo). Catch-all `/*` s manuálním parsováním je jediné funkční řešení (stejný pattern jako Maven adapter v1.1-07).

**SSRF ochrana v module path:**
Module path je user-controlled (z request URL). Před konstrukcí upstream URL:
- Validovat proti Go module path specifikaci (žádné `?`, `#`, `%00`, control characters)
- Použít `url.PathEscape()` pro upstream URL construction
- Odmítnout paths s `..` nebo absolutní paths

## Akceptační kritéria

1. `GOPROXY=http://shieldoo-gate:5005 go mod download github.com/user/repo@v1.0.0` uspěje pro čistý modul.
2. Malicious modul je blokován (HTTP 410 Gone — Go konvence pro blocked modules).
3. Metadata/list/mod requesty jsou proxovány bez scanningu.
4. Module paths s lomítky (github.com/user/repo) jsou správně routovány.
5. Module paths s uppercase (github.com/User/Repo) jsou správně encodovány (`!user/!repo`).
6. Major version suffix (github.com/user/repo/v2) je správně parsován.
7. Path traversal pokus vrátí HTTP 400.

## Rizika a mitigace

| Riziko | Dopad | Pravděpodobnost | Mitigace |
|--------|-------|-----------------|----------|
| Chi routing nefunkční pro slashed paths | Kritický — 404 pro všechny moduly | Jistý | Catch-all `/*` s manuálním parsováním |
| SSRF přes module path | Vysoký — internal access | Střední | Validace path; url.PathEscape; reject control chars |
| Bez sum.golang.org — tampered modules | Vysoký — nedetekováno | Nízká | Dokumentovat limitation; doporučit klientům ponechat GONOSUMDB |
| Module encoding bugs | Střední — cache collision | Nízká | Použít golang.org/x/mod/module (standard library) |
| GONOSUMCHECK requirement | Střední — user confusion | Střední | Dokumentovat v setup guide |

## Poznámky

- **golang.org/x/mod dependency:** Nová Go závislost pro module path encoding. Je to sub-module Go stdlib, minimální impact.
- **GONOSUMCHECK:** Klienti používající Shieldoo Gate jako GOPROXY mohou potřebovat `GONOSUMCHECK=*` pokud proxy modifikuje obsah (např. blokace). Dokumentovat v setup guide.
- **Dokumentační deliverables:** `docs/index.md`, `docs/adapters.md`, `docs/configuration.md`, setup guide s GOPROXY/GONOSUMCHECK.
