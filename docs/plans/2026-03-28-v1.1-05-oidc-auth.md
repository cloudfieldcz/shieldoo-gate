# v1.1 OIDC Admin Authentication — Technická analýza

## Popis

Přidání OIDC (OpenID Connect) autentizace pro admin API a UI. Proxy endpointy (PyPI, npm, NuGet, Docker) zůstávají bez autentizace — jsou chráněny síťovou segmentací.

### Proč

- Admin API (`port 8080`) je kompletně otevřený — kdokoli se síťovým přístupem může karanténovat/uvolnit artefakty, vytvářet override, triggerovat rescan
- `initial-analyse.md:1039` uvádí "Admin UI protected by authentication (local users or OIDC/SAML)"
- Security-critical akce (quarantine, release, override) vyžadují identifikaci operátora pro audit trail
- Enterprise prostředí vyžadují SSO integraci

### Out of Scope

- SAML — pouze OIDC v1.1
- Lokální uživatelé (username/password) — pouze OIDC. **Limitace:** Prostředí bez OIDC providera musí běžet s `auth.enabled: false`. Lokální auth plánována pro budoucí verzi.
- Autorizace (RBAC) — v1.1 má binární auth: autentizovaný = plný přístup
- Autentizace proxy endpointů (PyPI/npm/NuGet/Docker) — chráněno sítí

## Návrh řešení

### Architektura

```
Browser/API client
    │
    │ GET /api/v1/artifacts
    │ Authorization: Bearer <JWT>
    ▼
chi Router (server.go)
    │
    ├── /api/v1/health       → NO auth (health check)
    ├── /metrics             → **Separátní port (9090)** nebo basic auth/IP allowlist — Prometheus metriky mohou leakovat operační data (artifact counts, scanner error rates, internal hostnames)
    │
    └── /api/v1/*            → OIDC middleware
        │                       │
        │                  Validate JWT
        │                  (signature, expiry, issuer, audience)
        │                       │
        │                  Extract claims → context
        │                       │
        └── Handler (with user identity in ctx)
```

**OIDC flow pro UI (Authorization Code + PKCE):**
```
1. User opens UI → redirected to OIDC provider login
2. Provider authenticates → redirect back with code
3. UI exchanges code for tokens (via backend /auth/callback)
4. UI stores access token, sends as Bearer header
5. Backend validates JWT on each request
```

**OIDC flow pro API (machine-to-machine):**
```
1. Client obtains token from OIDC provider (client credentials grant)
2. Client sends: Authorization: Bearer <token>
3. Backend validates JWT
```

### OIDC Middleware

```go
// internal/auth/oidc.go

type OIDCConfig struct {
    IssuerURL    string   // https://accounts.google.com
    ClientID     string   // from config
    ClientSecret string   // from env var
    RedirectURL  string   // https://shieldoo-gate:8080/auth/callback
    Scopes       []string // ["openid", "email", "profile"]
}

// Middleware returns chi middleware that validates JWT Bearer tokens.
func Middleware(cfg OIDCConfig) func(http.Handler) http.Handler
```

JWT validace:
- Stáhnout JWKS z `issuer_url/.well-known/openid-configuration`
- Ověřit podpis (RS256/ES256)
- Ověřit `exp`, `iss`, `aud` claims
- Extrahovat `sub`, `email`, `name` → uložit do `context.Context`
- **`aud` claim validace je povinná** — `aud == client_id`. Bez toho tokeny vydané pro jiné aplikace od stejného IdP mohou být zneužity.
- **CSRF ochrana:** Pro cookie-based auth: `SameSite=Strict` + CSRF token. Pro Bearer token z localStorage: striktní CORS (žádný wildcard origin).

### Auth handlers (UI flow)

```go
// internal/auth/handlers.go

// GET /auth/login — redirect to OIDC provider
// HandleLogin generates OAuth2 state parameter stored in session (CSRF prevention).
func HandleLogin(cfg OIDCConfig) http.HandlerFunc

// GET /auth/callback — exchange code for tokens
// HandleCallback validates state parameter before exchanging code.
func HandleCallback(cfg OIDCConfig) http.HandlerFunc

// GET /auth/userinfo — return current user info
func HandleUserInfo() http.HandlerFunc

// POST /auth/logout — clear session
func HandleLogout() http.HandlerFunc
```

### Session management — rozhodnutí

**Zvoleno: httpOnly + Secure + SameSite=Strict cookies s CSRF tokenem.**

Důvody:
- httpOnly cookies jsou chráněny proti XSS (JavaScript nemůže číst token)
- SameSite=Strict brání CSRF útokům z cross-origin stránek
- Explicitní CSRF token jako double-submit cookie pro extra ochranu
- localStorage alternativa je zranitelná vůči XSS (security-critical admin interface to vylučuje)

**Token lifecycle:**
- Access token: 15 minut expiry
- Refresh token: 24 hodin, uložen v httpOnly cookie
- Silent renewal: UI volá `/auth/refresh` před expirací access tokenu
- Při selhání refresh: redirect na `/auth/login`

**Token revocation:**
- `POST /auth/logout` přidá `jti` (JWT ID) do in-memory blacklistu (TTL = token max lifetime)
- Blacklist je per-instance (nesdílený); pro HA: krátký token expiry (15min) minimalizuje window

### Změny v server.go

```go
func (s *Server) Routes() http.Handler {
    r := chi.NewRouter()
    r.Use(middleware.Recoverer)
    r.Use(middleware.RealIP)

    // Unauthenticated routes
    r.Get("/api/v1/health", s.handleHealth)
    // Metrics na separátním portu (9090) — nesmí být na admin portu bez auth
    // pokud auth.enabled: metriky obsahují operační data

    // Auth endpoints (OIDC flow)
    if s.authCfg.Enabled {
        r.Get("/auth/login", auth.HandleLogin(s.authCfg))
        r.Get("/auth/callback", auth.HandleCallback(s.authCfg))
        r.Get("/auth/userinfo", auth.HandleUserInfo())
        r.Post("/auth/logout", auth.HandleLogout())
    }

    // Authenticated API routes
    r.Group(func(r chi.Router) {
        if s.authCfg.Enabled {
            r.Use(auth.Middleware(s.authCfg))
        }
        r.Route("/api/v1", func(r chi.Router) {
            // ... existing artifact, audit, override, stats, docker routes
        })
    })

    // SPA fallback (UI assets)
    r.Get("/*", s.handleSPA)

    return r
}
```

## Akceptační kritéria

1. S `auth.enabled: true`: API endpointy (mimo health/metrics) vyžadují validní Bearer JWT.
2. S `auth.enabled: false`: chování identické jako dnes (žádná auth).
3. OIDC login flow funguje s Google, Azure AD, Keycloak.
4. Audit log obsahuje `user_email` pro autentizované akce.
5. Health endpoint (`/api/v1/health`) je vždy bez auth.
6. Metrics endpoint je na separátním portu (default 9090), ne na admin API portu.
7. Invalid/expired JWT → HTTP 401 s JSON error.
8. Proxy endpointy (PyPI/npm/NuGet/Docker porty) nejsou ovlivněny.
9. CSRF ochrana: POST/PUT/DELETE endpointy vyžadují validní CSRF token (double-submit cookie).
10. `aud` claim v JWT musí odpovídat `client_id` — jinak HTTP 401.
11. `auth.enabled: false` loguje WARNING při startu.
12. Token refresh: UI získá nový access token přes refresh token bez re-loginu.
13. `POST /auth/logout` invaliduje token (in-memory blacklist po dobu token lifetime).

## Rizika a mitigace

| Riziko | Dopad | Pravděpodobnost | Mitigace |
|--------|-------|-----------------|----------|
| OIDC provider nedostupný | Vysoký — admin API nefunkční | Nízká | JWT validace offline (JWKS cache); provider outage = nové tokeny nejdou, existující fungují |
| JWT cache invalidation | Střední — odvolaný token stále platný | Střední | Krátký token expiry (15min); introspection endpoint pro kritické akce (budoucí) |
| Client secret leak | Kritický | Nízká | Env var reference, nikdy v config souboru |
| Breaking change pro API klienty | Střední — existující skripty přestanou fungovat | Střední | `enabled: false` default; dokumentace migrace |
| CSRF útok na quarantine/release | Kritický — manipulace artefaktů přes prohlížeč | Střední | SameSite=Strict cookies + CSRF double-submit token |
| Metrics leakují operační data | Střední — topology disclosure | Střední | Separátní port (9090) |
| Auth disabled by default | Vysoký — open admin API v produkci | Střední | Prominent startup WARNING; no-default-credentials |

## Poznámky

- **Proxy endpointy zůstávají bez auth** — `pip install`, `docker pull` atd. nemohou posílat OIDC tokeny. Proxy je chráněn síťovou segmentací (interní síť).
- **JWKS caching:** OIDC knihovna (go-oidc) cachuje JWKS klíče. Při rotaci klíčů se automaticky refreshne.
- **Zpětná kompatibilita:** 100% — auth je opt-in (`enabled: false` default).
- **Nové Go závislosti** (`go-oidc`, `oauth2`) nejsou v approved list v CLAUDE.md. Vyžadují ADR.
- **Dokumentační deliverables:** `docs/index.md`, `docs/configuration.md`, `docs/api/openapi.yaml` (auth endpointy: /auth/login, /auth/callback, /auth/userinfo, /auth/logout, /auth/refresh).
