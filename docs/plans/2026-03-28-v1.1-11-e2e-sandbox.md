# v1.1 E2E Test Sandbox (Containerized Clients) — Technická analýza

## Popis

Kontejnerizace E2E test klientů (uv, npm, dotnet, maven, gem, go) tak, aby E2E testy neměly žádnou závislost na host-installed toolchainech a nebyly ovlivněny lokálními package-manager cache (~/.npm, ~/.nuget).

### Proč

- Současné E2E testy vyžadují `pip`/`uv`, `npm`, `dotnet` nainstalované na hostu
- Lokální cache interference: ~/.npm, ~/.nuget mohou maskovat chyby
- CI/CD reproducibilita: test výsledky závisí na host OS a instalovaných verzích
- `initial-analyse.md:1079` definuje E2E test sandbox jako v1.1
- Nové adaptéry (Maven, RubyGems, Go) vyžadují odpovídající test klienty

### Out of Scope

- Performance testy
- Load testy

## Aktuální stav

### E2E setup

`tests/e2e-shell/docker-compose.e2e.yml`:
- Shieldoo Gate + scanner bridge + push registry kontejnerizovány
- **Test klienty NEJSOU kontejnerizovány** — shell skripty (`test_pypi.sh`, `test_npm.sh`, `test_nuget.sh`, `test_docker.sh`) běží na hostu

`tests/e2e/e2e_test.go`:
- Go E2E testy s build tagem `//go:build e2e`
- Volají `pip install`, `npm install` přímo
- Předpokládají host-installed toolchains

`Makefile:14-15`:
```makefile
test-e2e:
    go test -tags e2e ./tests/e2e/... -v -count=1
```

| Aspekt | Současný stav | Navrhovaný stav |
|--------|--------------|-----------------|
| Test klienty | Host-installed | Docker kontejnery |
| Závislosti | pip, npm, dotnet na hostu | Pouze Docker |
| Cache izolace | Žádná | Izolovaný kontejner |
| Reproducibilita | Závisí na hostu | Deterministic |

## Návrh řešení

### Architektura

```
docker-compose.e2e.yml
    │
    ├── shieldoo-gate         (existující)
    ├── scanner-bridge        (existující)
    ├── push-registry         (existující)
    │
    └── test-runner           (NOVÝ)
        ├── uv / pip
        ├── npm
        ├── dotnet
        ├── mvn (Maven)
        ├── gem (RubyGems)
        ├── go
        └── docker CLI
```

### Test Runner kontejner

```dockerfile
# tests/e2e-shell/Dockerfile.test-runner
FROM ubuntu:24.04

# Python (uv)
RUN apt-get update && apt-get install -y curl
RUN curl -LsSf https://astral.sh/uv/install.sh | sh

# Node.js (npm)
RUN curl -fsSL https://deb.nodesource.com/setup_20.x | bash - && \
    apt-get install -y nodejs

# .NET (dotnet)
RUN apt-get install -y dotnet-sdk-8.0

# Maven
RUN apt-get install -y maven

# Ruby (gem)
RUN apt-get install -y ruby

# Go
RUN curl -fsSL https://go.dev/dl/go1.25.linux-amd64.tar.gz | tar -C /usr/local -xzf -
ENV PATH="/usr/local/go/bin:${PATH}"

# Docker CLI (pro Docker E2E testy)
RUN apt-get install -y docker.io

WORKDIR /tests
COPY tests/e2e-shell/ /tests/
```

### Docker Compose integrace

```yaml
# tests/e2e-shell/docker-compose.e2e.yml — rozšíření

test-runner:
  build:
    context: ../..
    dockerfile: tests/e2e-shell/Dockerfile.test-runner
  depends_on:
    shieldoo-gate:
      condition: service_healthy
  environment:
    - SGW_PYPI_URL=http://shieldoo-gate:5000
    - SGW_NPM_URL=http://shieldoo-gate:4873
    - SGW_NUGET_URL=http://shieldoo-gate:5001
    - SGW_DOCKER_URL=http://shieldoo-gate:5002
    - SGW_ADMIN_URL=http://shieldoo-gate:8080
    - SGW_MAVEN_URL=http://shieldoo-gate:5003
  volumes:
    - /var/run/docker.sock:/var/run/docker.sock  # pro Docker CLI
  networks:
    - default
```

### Test skripty

Stávající shell skripty se upraví tak, aby běžely uvnitř kontejneru:
- `test_pypi.sh` — `uv pip install --index-url $SGW_PYPI_URL ...`
- `test_npm.sh` — `npm install --registry $SGW_NPM_URL ...`
- `test_nuget.sh` — `dotnet nuget add source $SGW_NUGET_URL ...`
- `test_maven.sh` — **NOVÝ** — `mvn dependency:resolve -DremoteRepositories=$SGW_MAVEN_URL`
- `test_rubygems.sh` — **NOVÝ** — `gem install --source $SGW_RUBYGEMS_URL ...`
- `test_gomod.sh` — **NOVÝ** — `GOPROXY=$SGW_GOMOD_URL go mod download ...`

### Makefile target

```makefile
test-e2e-containerized:
    docker compose -f tests/e2e-shell/docker-compose.e2e.yml up --build --abort-on-container-exit test-runner
    docker compose -f tests/e2e-shell/docker-compose.e2e.yml down -v
```
