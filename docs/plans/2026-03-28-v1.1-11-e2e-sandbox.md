# v1.1 E2E Test Sandbox (Containerized Clients) — Technická analýza

## Popis

Kontejnerizace E2E test klientů (uv, npm, dotnet, maven, gem, go) tak, aby E2E testy neměly žádnou závislost na host-installed toolchainech a nebyly ovlivněny lokálními package-manager cache (~/.npm, ~/.nuget).

### Proč

- Současné E2E testy vyžadují `pip`/`uv`, `npm`, `dotnet` nainstalované na hostu
- Lokální cache interference: ~/.npm, ~/.nuget mohou maskovat chyby
- CI/CD reproducibilita: test výsledky závisí na host OS a instalovaných verzích
- `initial-analyse.md:1079` definuje E2E test sandbox jako v1.1
- Nové adaptéry (Maven, RubyGems, Go) vyžadují odpovídající test klienty

### Out of Scope

- Performance testy
- Load testy

## Aktuální stav

### E2E setup

`tests/e2e-shell/docker-compose.e2e.yml`:
- Shieldoo Gate + scanner bridge + push registry kontejnerizovány
- **Test klienty NEJSOU kontejnerizovány** — shell skripty (`test_pypi.sh`, `test_npm.sh`, `test_nuget.sh`, `test_docker.sh`) běží na hostu

`tests/e2e/e2e_test.go`:
- Go E2E testy s build tagem `//go:build e2e`
- Volají `pip install`, `npm install` přímo
- Předpokládají host-installed toolchains

`Makefile:14-15`:
```makefile
test-e2e:
    go test -tags e2e ./tests/e2e/... -v -count=1
```

| Aspekt | Současný stav | Navrhovaný stav |
|--------|--------------|-----------------|
| Test klienty | Host-installed | Docker kontejnery |
| Závislosti | pip, npm, dotnet na hostu | Pouze Docker |
| Cache izolace | Žádná | Izolovaný kontejner |
| Reproducibilita | Závisí na hostu | Deterministic |

### Test Runner kontejner

```dockerfile
# tests/e2e-shell/Dockerfile.test-runner
FROM ubuntu:24.04

# VŠECHNY VERZE PINOVANÉ (dle CLAUDE.md)

# Python (uv) — pinovaná verze
ARG UV_VERSION=0.6.0
RUN apt-get update && apt-get install -y curl ca-certificates && \
    curl -LsSf "https://github.com/astral-sh/uv/releases/download/${UV_VERSION}/uv-installer.sh" | \
    sh -s -- --version ${UV_VERSION}

# Node.js — pinovaná verze
ARG NODE_VERSION=20.19.0
RUN curl -fsSL https://deb.nodesource.com/setup_20.x | bash - && \
    apt-get install -y nodejs=${NODE_VERSION}-1nodesource1

# .NET SDK — pinovaná verze
ARG DOTNET_VERSION=8.0
RUN apt-get install -y wget && \
    wget https://packages.microsoft.com/config/ubuntu/24.04/packages-microsoft-prod.deb && \
    dpkg -i packages-microsoft-prod.deb && \
    apt-get update && apt-get install -y dotnet-sdk-${DOTNET_VERSION}

# Maven — pinovaná verze
ARG MAVEN_VERSION=3.9.6
RUN apt-get install -y maven=${MAVEN_VERSION}*

# Ruby — pinovaná verze
RUN apt-get install -y ruby

# Go — pinovaná verze s checksum
ARG GO_VERSION=1.25.0
ARG GO_SHA256=...
RUN curl -fsSL "https://go.dev/dl/go${GO_VERSION}.linux-amd64.tar.gz" -o go.tar.gz && \
    echo "${GO_SHA256}  go.tar.gz" | sha256sum -c - && \
    tar -C /usr/local -xzf go.tar.gz && rm go.tar.gz
ENV PATH="/usr/local/go/bin:${PATH}"

WORKDIR /tests
COPY tests/e2e-shell/ /tests/
```

### Docker Compose integrace

```yaml
# tests/e2e-shell/docker-compose.e2e.yml — rozšíření

test-runner:
  build:
    context: ../..
    dockerfile: tests/e2e-shell/Dockerfile.test-runner
  depends_on:
    shieldoo-gate:
      condition: service_healthy
  environment:
    - SGW_PYPI_URL=http://shieldoo-gate:5000
    - SGW_NPM_URL=http://shieldoo-gate:4873
    - SGW_NUGET_URL=http://shieldoo-gate:5001
    - SGW_DOCKER_URL=http://shieldoo-gate:5002
    - SGW_ADMIN_URL=http://shieldoo-gate:8080
    - SGW_MAVEN_URL=http://shieldoo-gate:5003
  # BEZPEČNOST: NIKDY nemontovat Docker socket — umožňuje container escape.
  # Pro Docker E2E testy použít Docker-in-Docker (DinD):

  docker-dind:
    image: docker:24.0-dind
    privileged: true
    environment:
      - DOCKER_TLS_CERTDIR=""
    networks:
      - default

  test-runner:
    # ...
    environment:
      - DOCKER_HOST=tcp://docker-dind:2375
    # BEZ volume mount na docker.sock!
```

### Síťová izolace

Test runner by měl mít přístup pouze k proxy portům, ne k admin API. Pro admin API testy použít separátní kontejner s omezeným přístupem.

```yaml
networks:
  proxy-net:    # test-runner + shieldoo-gate proxy ports
  admin-net:    # admin-test-runner + shieldoo-gate admin port
  internal-net: # shieldoo-gate + scanner-bridge + dind
```

### Test skripty

Stávající shell skripty se upraví tak, aby běžely uvnitř kontejneru:
- `test_pypi.sh` — `uv pip install --index-url $SGW_PYPI_URL ...`
- `test_npm.sh` — `npm install --registry $SGW_NPM_URL ...`
- `test_nuget.sh` — `dotnet nuget add source $SGW_NUGET_URL ...`
- `test_maven.sh` — **NOVÝ** — `mvn dependency:resolve -DremoteRepositories=$SGW_MAVEN_URL`
- `test_rubygems.sh` — **NOVÝ** — `gem install --source $SGW_RUBYGEMS_URL ...`
- `test_gomod.sh` — **NOVÝ** — `GOPROXY=$SGW_GOMOD_URL go mod download ...`

## Akceptační kritéria

1. Všechny E2E testy projdou uvnitř kontejneru s nulovou závislostí na host-installed toolchainech.
2. Stávající host-based E2E testy (`make test-e2e`) fungují jako fallback.
3. `make test-e2e-containerized` exituje 0 na čistém Shieldoo Gate.
4. Docker E2E testy používají DinD (ne Docker socket mount).
5. Všechny tool verze v Dockerfile.test-runner jsou pinované s explicit verzí.
6. `run_all.sh` spustí všechny testy a exituje s non-zero pokud jakýkoli test selže.

## Testování

- `make test-e2e-containerized` — spustí všechny E2E testy v kontejneru
- Ověřit, že žádný test nepotřebuje host-installed toolchain
- CI: test-runner kontejner se buildí a spouští v CI pipeline

## Rizika a mitigace

| Riziko | Dopad | Pravděpodobnost | Mitigace |
|--------|-------|-----------------|----------|
| DinD privileged mode | Střední — privileged kontejner | N/A (test-only) | Pouze v E2E test compose; nikdy v produkci |
| Pinned version update burden | Nízký — maintenance | Střední | Dependabot/Renovate pro Dockerfile ARGs |
| Test runner network přístup k admin API | Střední — malicious package může manipulovat artefakty | Nízká | Separátní Docker networks |

## Poznámky

- **`run_all.sh` chování:** Spustí všechny test skripty sekvenčně. Exituje s non-zero pokud jakýkoli test selže. Individuální výsledky logované do stdout.
