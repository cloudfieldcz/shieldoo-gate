# v1.1 Maven Adapter — Technická analýza

## Popis

Nový protokolový adaptér pro Maven/Gradle repozitáře. Implementuje Maven repository HTTP layout jako transparent proxy s artifact scanning.

### Proč

- Java/Kotlin ekosystém je masivní — Maven Central má 12M+ artefaktů
- Supply chain útoky na Maven: dependency confusion, typosquatting (např. "log4j" typos)
- `initial-analyse.md:591-594` definuje Maven adapter jako v1.1
- Gradle (nejpopulárnější Java build tool) používá Maven repository formát

### Out of Scope

- Gradle-specific protokol (Gradle Module Metadata) — pouze Maven repository layout
- Maven deploy (upload) — pouze proxy/read

## Aktuální stav

### Adapter pattern

Všechny existující adaptéry (PyPI, npm, NuGet) sdílejí identický pattern:
1. Constructor: `New*Adapter(db, cacheStore, scanEngine, policyEngine, upstreamURL)`
2. Chi router s ecosystem-specific routes
3. `downloadScanServe()` pipeline: download → scan → policy → persist → serve/block
4. Cache hit path: check status → serve if CLEAN
5. Artifact ID: `"{ecosystem}:{name}:{version}"`

### Ecosystem constants

- `EcosystemMaven Ecosystem = "maven"` — **již definováno!**

### Scanner support

- GuardDog: nepodporuje Maven (pouze PyPI, npm)
- Trivy: podporuje Maven (filesystem scan)
- OSV: podporuje Maven (API lookup)
- Built-in scanners: většina generic (hash verifier, obfuscation, exfil)

## Návrh řešení

### Maven Repository HTTP Layout

```
GET /{group}/{artifact}/{version}/{artifact}-{version}.jar       ← download JAR
GET /{group}/{artifact}/{version}/{artifact}-{version}.pom       ← POM metadata
GET /{group}/{artifact}/{version}/{artifact}-{version}.jar.sha1  ← checksum
GET /{group}/{artifact}/maven-metadata.xml                        ← version listing
```

Group ID je path-encoded: `org.apache.commons` → `org/apache/commons`

### Routing

```go
r.Get("/*", a.handleRequest)  // catch-all, parse path to determine action
```

Maven layout je path-based, ne route-based. Handler parsuje cestu:
- `*.jar` / `*.war` / `*.aar` → artifact download (trigger scan)
- `*.pom` → POM metadata (proxy pass-through, no scan)
- `*.sha1` / `*.md5` / `*.sha256` → checksum (proxy pass-through)
- `maven-metadata.xml` → version listing (proxy pass-through)

### Artifact ID format

`maven:org.apache.commons:commons-lang3:3.14.0`

Group ID + artifact ID + version, separátor `:`.

## Rizika a mitigace

| Riziko | Dopad | Pravděpodobnost | Mitigace |
|--------|-------|-----------------|----------|
| Komplexní Maven path parsing | Střední — edge cases | Střední | Extensive unit testy s reálnými Maven paths |
| JAR obfuscation (bytecode) | Nízký — scanner limitace | Střední | Trivy + OSV pokryjí známé CVE; bytecode analysis budoucí |
